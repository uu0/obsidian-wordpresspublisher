# 信息架构重组方案

## 当前结构分析

### 现有三标签页结构

#### **Settings 标签**
- ✅ 特色图片设置（Vault/Unsplash/AI 生成）
- ✅ 基础设置（标题、分类、标签、状态、日期等）

#### **Preview 标签**
- ✅ 内容预览
- ✅ 编辑功能
- ✅ AI 生成摘要按钮（在底部栏）

#### **Advanced 标签**
- ⚠️ AI Prompt 设置（摘要、标签、图片生成）
- ⚠️ 保存设置按钮

---

## 问题分析

### 1. **信息架构混乱**
- AI 生成功能分散在两个标签页（Preview 和 Advanced）
- Advanced 标签实际上是"AI Prompt 设置"，但名称不直观
- 用户需要在 Preview 和 Advanced 之间来回切换才能完成 AI 相关操作

### 2. **操作流程不顺畅**
```
当前流程：
Settings → 填写基础信息
Preview → 查看内容 → 点击"生成摘要"
Advanced → 修改 AI Prompt → 保存设置
Preview → 再次生成摘要
```

### 3. **高级设置位置不当**
- AI Prompt 设置应该是全局配置，不应该在发布模态框中
- 每次发布都显示这些设置会增加认知负担

---

## 重组方案

### **新的两标签页结构**

#### **📝 Settings 标签**（编辑和配置）

```
┌─────────────────────────────────────────┐
│ 📝 Settings                             │
├─────────────────────────────────────────┤
│                                         │
│ ┌─── 特色图片 ───────────────────────┐ │
│ │ [预览图]                            │ │
│ │ [从 Vault 选择] [Unsplash] [AI生成] │ │
│ └─────────────────────────────────────┘ │
│                                         │
│ ┌─── 基础设置 ───────────────────────┐ │
│ │ 标题: [___________]                 │ │
│ │ 分类: [下拉选择]                    │ │
│ │ 标签: [标签1] [标签2] [+]           │ │
│ │ 状态: [发布/草稿]                   │ │
│ │ 日期: [2026-03-14]                  │ │
│ └─────────────────────────────────────┘ │
│                                         │
│ ┌─── AI 辅助 ─────────────────────────┐ │
│ │ 📊 生成摘要                          │ │
│ │ [生成] [重新生成]                    │ │
│ │                                      │ │
│ │ 🏷️ 智能标签推荐                      │ │
│ │ [生成标签] → [推荐1] [推荐2] [推荐3] │ │
│ │                                      │ │
│ │ 🎨 AI 生成特色图片                   │ │
│ │ [生成图片] [查看历史]                │ │
│ └─────────────────────────────────────┘ │
│                                         │
└─────────────────────────────────────────┘
```

**特点：**
- 所有编辑和配置功能集中在一个标签页
- AI 功能作为"辅助工具"区域，清晰分组
- 操作流程更顺畅：填写 → AI 辅助 → 发布

- 移除 AI 生成按钮（已移到 Settings）

- 发布按钮更突出

## 实施计划

### Phase 1: 重构标签页结构
1. 移除 `renderAdvancedTab()` 方法
2. 在 `renderSettingsTab()` 中添加 AI 辅助区域
3. 简化 `renderPreviewTab()`，移除 AI 按钮

### Phase 2: 迁移 AI Prompt 设置
1. 在插件设置页面添加"AI 设置"部分
2. 移动 Prompt 编辑器到设置页面
3. 更新设置保存逻辑

### Phase 3: 优化交互流程
1. 添加 AI 生成进度指示器
2. 优化标签推荐 UI（可点击添加）
3. 添加发布选项（发布后打开、更新 Frontmatter）

## 兼容性考虑

### 设置迁移
```typescript
// 在插件加载时自动迁移
async onload() {
  await this.loadSettings();

  // 迁移旧的 Advanced 标签设置到插件设置
  if (!this.settings.aiPromptsVersion) {
    this.settings.aiPromptsVersion = '2.0';
    await this.saveSettings();
  }
}
```

### 用户通知
```typescript
// 首次打开新版本时显示提示
if (this.settings.lastVersion !== '2.0') {
  new Notice('📢 WordPress Publisher 已更新！AI 设置已移至插件设置页面。', 5000);
  this.settings.lastVersion = '2.0';
  await this.saveSettings();
}
```
